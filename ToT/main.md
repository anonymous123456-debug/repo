# ToT Baseline

ToT replaces direct one-shot answer generation with an iterative search process: generate candidate thoughts, evaluate candidates, select candidates, and continue expanding. This document introduces how to run the baseline script.

## 1. Environment Setup

```bash
cd /path/to/ToT
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

When preparing data, place files under `src/tot/data/` according to the relative paths used in the code. For example:

- `src/tot/data/hotpotqa/hotpotqa.json`

## 2. Quick Start

The generic runner is recommended:

```bash
cd /path/to/ToT
export TOT_MODEL_PATH="YOUR_LOCAL_OR_HF_MODEL_PATH"
bash sh/main.sh
```

The default configuration runs:

```text
TASK=winograd
MODE=tot
TASK_START_INDEX=0
TASK_END_INDEX=200
BACKEND=local
METHOD_GENERATE=propose
METHOD_EVALUATE=value
METHOD_SELECT=greedy
N_GENERATE_SAMPLE=20
N_EVALUATE_SAMPLE=3
N_SELECT_SAMPLE=3
STEP=3
```

To switch the dataset or running mode:

```bash
TASK=commonsenseqa MODE=tot bash sh/main.sh
```





## 3. Main Parameters

- `TASK`: Dataset task name. Default: `winograd`.
- `MODE`: Running paradigm. Default: `tot`.
- `TASK_START_INDEX`: Starting sample index. Default: `0`.
- `TASK_END_INDEX`: Ending sample index. Default: `200`.
- `TOT_MODEL_PATH`: Local model path or Hugging Face model ID.
- `STEP`: ToT search depth. Default: `3`.
- `METHOD_GENERATE`: Candidate generation method for ToT. Default: `propose`.
- `METHOD_EVALUATE`: Candidate evaluation method for ToT. Default: `value`.
- `METHOD_SELECT`: Candidate selection method for ToT. Default: `greedy`.
- `N_GENERATE_SAMPLE`: Number of candidates generated at each step. Default: `20`.
- `N_EVALUATE_SAMPLE`: Number of evaluation samples for candidate scoring. Default: `3`.
- `N_SELECT_SAMPLE`: Number of candidates retained at each step, i.e. the search width. Default: `3`.

## 4. Execution Flow

1. `sh/main.sh` sets the project root, `PYTHONPATH`, model path, and runtime parameters.
2. `run.py` calls `get_task()` to load the selected task.
3. The task class reads data from `src/tot/data/` and limits execution to the first 200 samples or the available sample count.
4. In ToT mode, `solve()` runs candidate generation, candidate scoring, candidate selection, and final answer generation.
5. In naive mode, `naive_solve()` directly samples answers with the standard or CoT prompt.
6. The task class computes accuracy or F1 through `test_output()`.
7. Results are written to a JSON file under `logs/<task>/`.

## 5. Outputs

Run results are saved to:

```text
logs/<task>/<run_config>.json
```

Each record usually contains:

- `idx`: Sample index.
- `ys`: Answers generated by the model.
- `infos`: Evaluation results for the current sample.
- `steps`: Intermediate candidates, scores, and selections from the ToT search process.
- `usage_so_far`: Current cumulative token statistics.
